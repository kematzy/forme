Bootstrap 3 Forms Syntax - For Forme

Bootstrap syntax examples

The following examples present the expected Bootstrap 3 formatting for each type of inputs, and especially for the :checkboxset and :radioset.

Text input - <input type="text">

Default: (without Error)

<div class="form-group">
  <label for="name">Name</label>
  <input class="form-control" id="name" name="name" type="text"/>
</div>

With Error:
- CSS class .has-error added to .form-group wrapper div
- error message with CSS class .help-block .with-errors added wrapper div.

Error for input[:text] 
<div class="form-group has-error">
  <label for="name">Name</label> 
  <input class="form-control error" id="name" name="name" type="text"/>
  <span class="help-block with-errors">Error for input[:text]</span>
</div>

Checkbox / Radio Sets


Please Note!

CheckboxSets and RadioSets are Forme options translated into Bootstrap 3 HTML output.

Since Bootstrap 3 does not clearly define how to handle error output for CheckboxSets / RadioSets, I have presented two output variants below:

Alternative A: (current Forme output)
presents the individual :checkbox/:radio tags without any error formatting and message, and instead adds one error message below the tags within the div.has-error wrapper.

Alternative B:
presents each individual :checkbox/:radio with the same formatting as a single :checkbox/:radio would be presented, ie: with a div.has-error wrapper and an error message within that wrapper.

I believe Alternative A is the correct output, but I am not sure, so your input and opinion is most welcome!

(Only one variant should be supported by Forme)

Radio Set - @f.input(:radioset)

Please Note! This is really a Forme option that is being translated into BS3 format.
I have therefore provided two (2) alternative output variants, but only one should be supported in Forme's codebase.

Alternative A: is probably the easiest one to support, but not a very elegant solution, which also requires additional CSS support.

Alternative B: is the more elegant version, but may be more difficult to support in Forme.

Your input and opinion is sought and most welcome!
